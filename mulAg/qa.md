# qa.md

## review 검토 규칙

- review와 실제 파일 상태(Explorer)를 비교한다.
- 변경 파일/생성 파일이 todo와 일치하는지 확인한다.
- todo에 없는 파일 변경은 실패 사유로 처리한다.

## Explorer 검토 기준

- 변경 파일 일치성
- 생성 파일 일치성
- 읽기 전용/수정 금지 파일 변경 여부
- 파일 권한 위반 여부
- todo 완료 기준 충족 여부
- 모듈 경계 및 정합성 점검

## done 처리 기준

다음 모두 충족 시 done 처리.

- todo 상태가 review 완료 및 승인 준비
- 완료 기준 충족
- 검증 증거(명령/로그) 존재
- 위반 항목 없음

## plan 완료/보류/폐기/대체 판단

- 모든 관련 todo가 done/rejected 상태로 정리되면 done
- 선행 조건 미충족은 blocked 또는 deferred
- 요구사항 상충 시 canceled/superseded

## 반려 기준

- 완료 기준 미충족
- 파일 권한 위반
- 선행 조건 미충족
- 책임 경계/모듈 경계 침범
- 정합성 훼손
- 검증 부재

## 파일 권한 위반 검토

- todo에서 선언한 수정/생성 범위 외 변경 여부 확인
- 공통 영향 파일 변경 시 별도 승인 요구

## 선행 조건 검토

- 선행 todo, 선행 review 또는 plan 완료 여부 확인
- 미완료면 해당 todo는 대기/차단 처리

## plan/todo 재작성 요청 기준

- 규칙 변경이 필요하면 새 plan 또는 후속 todo로 분리한다.
- 기존 plan/todo를 임의 덮어쓰지 않는다.
