# sub-agent.md

## todo 참조 규칙

- 먼저 `todo` 본문 전체를 읽는다.
- plan은 배경 참고용이므로 todo를 넘어서는 확장을 금지한다.
- todo 없이 임의 변경을 수행하지 않는다.

## 작업 수행 규칙

- 작업 시작 시 todo 상태를 `running`으로 간주해 진행한다.
- 지정 파일 범위 내에서만 수정한다.
- todo 완료 후 review 작성 후 todo 상태를 `review`로 둔다.

## 파일 수정 제한

- 수정 가능 파일: todo에서 명시한 목록만
- 생성 가능 파일: todo에서 명시한 목록만
- 읽기 전용 파일: 참고만 가능
- 수정 금지 파일: 읽기조차 허용되더라도 수정/삭제 금지

## 생성 가능 파일 제한

- 생성은 요구사항 수용 후 결과물(리뷰/문서/코드)을 최소 단위로 생성한다.
- 불필요한 구조물/임시파일은 만들지 않는다.

## 작업 범위 이탈 처리

- todo 범위를 벗어난 작업 필요 발생 시 즉시 review의 `남은 이슈`에 기록한다.
- 즉시 block 처리하지 않고 후속 todo 또는 plan 수정 요청으로 전환한다.

## review 작성 규칙

review는 다음 항목을 포함한다.

- 문서 ID
- 수행 일시
- 참조 plan / 참조 todo
- 수행 내용
- 변경 파일 / 생성 파일 / 미변경 파일
- 변경 요약(파일/유형/요약)
- 검증 내용(실행 명령 및 결과)
- 남은 이슈
- QA 확인 요청 사항
